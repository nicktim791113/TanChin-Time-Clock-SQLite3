# 快速部署還原說明

這份文件放在核心資產備份壓縮檔內，用來在新電腦、重灌後或資料搬移時快速恢復 TanChin-Time-Clock-SQLite3。

## 備份檔內容結構

解壓縮核心資產備份檔後，主要會看到：

```text
backup-manifest.json
project-source\
runtime-user-data\
```

- `project-source`：專案原始碼、開發文件、文案、備份腳本。
- `runtime-user-data`：實際營運資料，例如 `app_data.db`、自訂音效、主題圖片、LOG。
- `backup-manifest.json`：本次備份的來源、時間、包含項目與缺漏項目。

## 情境一：只把資料恢復到已安裝的軟體

適合情況：

- 你已經安裝好 TanChin-Time-Clock-SQLite3。
- 只想把員工資料、打卡資料、自訂音效、主題圖、設定資料恢復回來。

步驟：

1. 關閉打卡軟體。
2. 解壓縮備份 zip。
3. 複製備份檔內的：

```text
runtime-user-data\*
```

4. 貼到並覆蓋：

```text
%APPDATA%\TanChin-Time-Clock-SQLite3
```

通常完整路徑是：

```text
C:\Users\你的使用者名稱\AppData\Roaming\TanChin-Time-Clock-SQLite3
```

5. 重新開啟打卡軟體。

檢查重點：

- 員工資料是否存在。
- 打卡紀錄是否存在。
- 自訂音效與主題圖片是否能正常顯示或播放。
- 管理者/開發人員工作台是否能登入。

## 情境二：恢復成可開發或可重新打包的專案

適合情況：

- 換電腦後要繼續開發。
- 要重新打包安裝檔。
- 要把原始碼完整搬到另一個資料夾。

步驟：

1. 解壓縮備份 zip。
2. 建立或選擇你的專案資料夾，例如：

```text
C:\Users\你的使用者名稱\Desktop\Developer\TanChin-Time-Clock-SQLite3
```

3. 把備份檔內的：

```text
project-source\*
```

複製到專案資料夾。

4. 在專案資料夾執行：

```powershell
npm install
```

5. 啟動開發版：

```powershell
npm start
```

6. 需要重新產生安裝檔時執行：

```powershell
npm run build
```

注意：

- 備份檔不包含 `node_modules`，所以新環境要先跑 `npm install`。
- 備份檔不包含 `dist`，因為打包輸出可以重新產生。
- 如果是全新電腦，需要先安裝 Node.js。

## 快速備份指令是否只能在這台電腦使用？

不是只能在這台電腦使用，但它主要是為 Windows + 這個 Electron 專案設計。

這支指令可以在同一個專案的其他 Windows 電腦使用：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1"
```

腳本會自動判斷：

- 專案來源：以 `scripts\backup-core-assets.ps1` 所在位置往上一層作為專案根目錄。
- 軟體營運資料：預設讀取 `%APPDATA%\TanChin-Time-Clock-SQLite3`。
- 備份輸出位置：預設輸出到目前 Windows 使用者底下的 OneDrive 路徑。

如果另一台電腦的 OneDrive 路徑不同，請改用自訂輸出位置：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -DestinationRoot "D:\TanChin-Time-Clock-SQLite3 原始資料備份"
```

如果另一台電腦的軟體資料夾不同，請指定 `UserDataPath`：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -UserDataPath "C:\Users\你的使用者名稱\AppData\Roaming\TanChin-Time-Clock-SQLite3"
```

也可以兩個都指定：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -DestinationRoot "D:\備份\TanChin" -UserDataPath "C:\Users\你的使用者名稱\AppData\Roaming\TanChin-Time-Clock-SQLite3"
```

## 建議還原前先做一件事

如果目標電腦已經有舊資料，還原前先把這個資料夾整個複製一份保存：

```text
%APPDATA%\TanChin-Time-Clock-SQLite3
```

這樣如果覆蓋錯資料，還可以退回原本狀態。

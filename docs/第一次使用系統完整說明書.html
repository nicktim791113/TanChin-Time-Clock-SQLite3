1. 文件狀態

檢查結果：目前專案內沒有找到一份完整的「第一次使用系統」HTML 說明書。 既有文件只有 docs/快速部署還原說明.md 與 docs/更新紀錄.md， 所以本文件是依照現有系統證據新建立的第一次使用完整手冊。

本文件不是單純功能介紹，而是依照實務開通順序撰寫。第一次使用時，請照章節順序執行，避免先建立大量資料後才發現帳號、權限或備份策略沒有設定好。

2. 最高權限原則

本系統的最高權限不是單一帳號包辦全部工作，而是分成兩個核心權責：

系統管理者

獨立於員工資料之外，主要管理「誰可以用哪些角色登入」以及「管理者可操作哪些功能」。這是帳號與權限的最高入口。

開發人員

綁定員工帳號，負責系統密碼、自動化任務、操作稽核、完整資料庫備份還原、主功能頁籤排序、系統狀態與 API 目錄檢查。

第一次使用時請先登入系統管理者，建立可信任的開發人員與完整管理者帳號，再由開發人員設定備份、自動化與系統密碼。不要把日常員工帳號直接當成最高權限共用帳號。

3. 角色與預設帳密

角色 登入方式 預設資訊 第一次使用後必做
系統管理者 登入入口選「系統管理者」，使用獨立帳號與密碼。 system-admin / TC-SYS-0000 立即修改系統管理者帳號或密碼，並記錄在公司內部安全保存位置。
開發人員 使用員工工號 + 系統密碼，且該員工必須被授權開發人員角色。 系統密碼預設 0000 授權可信任員工為開發人員，登入後到系統設定修改系統密碼。
管理者 使用員工工號 + 管理者密碼，且該員工必須被授權管理者角色。 管理者密碼預設 TC5128 指定完整管理者或分權管理者，並在安全設定中修改管理者密碼。
員工 使用員工編號 + 卡號登入。 卡號來自員工資料欄位。 確認員工工號、姓名、部門、職稱、卡號、密碼與個人資料正確。
預設密碼只適合第一次開通。正式使用前，至少要完成三個密碼更換：系統管理者密碼、系統密碼、管理者密碼。

4. 第一次開通順序

  1. 啟動桌面系統或開發環境，確認 Express 服務正常運行在 3000 連接埠。
  2. 開啟網頁版入口：http://127.0.0.1:3000/browser/。
  3. 使用系統管理者預設帳號登入，先修改系統管理者帳號或密碼。
  4. 確認員工資料是否已存在。若沒有員工資料，先建立或匯入最基本的員工名冊。
  5. 在帳號權限管理中，指定至少一位完整管理者與一位開發人員。
  6. 使用開發人員登入，修改系統密碼，設定完整資料庫備份資料夾與保留份數。
  7. 使用管理者登入，修改管理者密碼，建立員工、班別、遠端打卡安全、請假與報表設定。
  8. 建立一筆測試員工登入與打卡紀錄，確認裝置綁定與 GPS 權限是否符合公司政策。
  9. 建立一次完整資料庫備份，確認備份檔與 manifest 可成功產生。
  10. 匯出一份考勤報表，確認欄位順序、請假紀錄與 ERP 預期格式相容。

5. 帳號權限管理

帳號權限可以由「系統管理者」或「開發人員」管理。系統管理者是最適合第一次開通時使用的入口，因為它不依賴任何員工帳號。

可授權角色

  • 員工：所有員工帳號預設都保留員工登入權限。
  • 管理者：授權後可使用管理者密碼登入，但可用功能仍受管理者權限模板或自訂權限限制。
  • 開發人員：授權後可使用系統密碼登入開發者工作台。

管理者權限模板

模板 用途
無管理者權限 員工只能使用員工工作台，不進入管理者功能。
完整管理者 可使用全部管理者功能，適合公司主要負責人或系統營運主管。
人資管理者 偏向人員資料、請假與報表查詢。
考勤管理者 偏向班別、手動補登、考勤報表與匯出。
報表檢視者 只查看報表，不做高風險異動。
系統外觀管理者 管理主畫面、問候語、響鈴、主題與視覺相關設定。
自訂權限 逐項勾選管理者功能，適合精細分工。
請至少保留一個系統管理者帳號、一個完整管理者帳號與一個開發人員帳號。不要刪除或取消最後一個可管理權限的入口。

6. 管理者基礎設定

完整管理者登入後，建議依序設定下列內容。管理者工作台的主功能頁籤包含人員資料、安全設定、班別設定、手動補登、考勤報表、請假管理、系統設定、響鈴設定與主題特效。

人員資料

  • 基本欄位包含工號、姓名、性別、部門、職稱、卡號、密碼、國籍、身分證字號、生日、到職日、離職日與備註。
  • 延伸欄位包含銀行帳號、手機、緊急聯絡人、聯絡電話、戶籍地址、通訊地址與家庭狀態。
  • 匯入前先整理 CSV 欄位，避免卡號重複或工號空白。

安全設定

  • 裝置綁定預設啟用。員工第一次用網頁版登入與打卡時，系統會記錄裝置資訊。
  • 遠端打卡可要求 GPS，預設最大 GPS 誤差為 300 公尺。
  • 員工更換手機或瀏覽器時，管理者可解除舊裝置綁定。

班別與補登

  • 先建立公司實際使用的班別名稱、上班時間與下班時間。
  • 手動補登屬於高風險操作，應只開給考勤或完整管理者，並保留稽核紀錄。

系統顯示、響鈴與主題

  • 系統設定可調整主標題、副標與登入頁說明文字。
  • 響鈴設定可管理排程、自訂聲音與響鈴歷史。
  • 主題特效可管理節日特效、主題排程與自訂主題圖片。

7. 考勤與請假設定

打卡來源

系統會保存桌面端、密碼、手動補登、API、瀏覽器與請假等來源。這些來源會影響稽核與報表判讀。

請假流程

  • 請假類型支援小時或天數單位，可設定是否需附件、是否扣假、是否給薪與啟用狀態。
  • 請假路徑以部門對應主管，主管以員工身份登入審核，管理者再進行終審。
  • 請假可支援半小時、半天、一天與跨時段情境，系統會保存開始、結束與時數。

考勤報表與 ERP

考勤匯出模板包含薪資系統、異常稽核、報表分析、薪資含請假明細、完整格式與自訂格式。若要讓薪資與請假資料搭配得好，建議使用「薪資含請假明細」或自訂格式，並保留以下欄位：

  • 工號、姓名、部門、職稱、日期、時間、班別、打卡類型、考勤狀態、來源。
  • 紀錄種類、假別、請假開始、請假結束、請假時數、請假單號。
系統已設計讓核准請假也能成為考勤報表中的可查核紀錄，避免當天沒有打卡就完全沒有紀錄，方便後續 ERP 或薪資系統提取。

8. 開發者維運工作台

開發者工作台是系統維運的核心。第一次使用時，請不要只做帳號授權，也要完成以下維運設定。

系統設定

  • 變更系統密碼。開發人員登入、管理者密碼變更與身份切換都會使用系統密碼做高風險驗證。
  • 視需要啟用開發人員身份切換，用來測試管理者或員工流程。一般員工與管理者登入頁不接受系統密碼作為萬用密碼。
  • 設定主功能頁籤排序。開發人員可以在開發者工作台調整管理者與開發者工作台的主功能頁籤全域排序，並可還原預設。
  • 管理帳號權限。開發人員可和系統管理者一樣調整員工可登入角色與管理者功能權限。

稽核與系統狀態

  • 操作稽核紀錄可查登入、權限、打卡失敗、備份還原、密碼變更等重要事件。
  • 稽核封存可設定保留天數，預設保留 180 天。
  • 系統狀態會顯示服務狀態、連接埠、版本、資料庫位置、資料量、備份狀態、API 數量與自動化狀態。

9. 完整資料庫備份與還原

完整資料庫備份是第一次使用後一定要設定的保護機制。備份會產生 SQLite 資料庫檔與 manifest，manifest 會記錄檔案大小與 SHA256，方便日後查核。

第一次備份設定

  1. 使用開發人員登入。
  2. 進入「匯出設定」的「完整資料庫備份與還原」。
  3. 設定完整資料庫備份資料夾。若未設定，系統會使用預設備份資料夾。
  4. 設定保留份數，預設為 30 份。
  5. 按下立即完整備份，確認最近完整備份清單出現新檔案。

還原規則

  • 還原前系統會先建立還原前緊急備份。
  • 來源檔需通過 SQLite 檢查與必要資料表檢查。
  • 執行還原前必須輸入確認文字：還原資料庫。
  • 還原會覆蓋目前使用中的資料庫，正式環境請先取得主管或系統負責人確認。

專案核心備份

除了網頁內的資料庫備份，專案也提供核心資產備份腳本，可打包原始碼、文件、使用者資料庫與自訂素材。

powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1"

10. 自動化任務

自動化任務目前可處理匯出、刪除與完整資料庫備份。第一次設定時建議先做「完整資料庫備份」排程，再設定報表匯出。

任務類型 可用目標 建議用途
匯出 上週打卡紀錄、上月打卡紀錄、手動補登紀錄、全部打卡紀錄、全部員工資料、全部響鈴紀錄、系統日誌。 固定產生考勤、員工或系統資料，交給 ERP 或內部稽核使用。
刪除 依系統允許的清理目標執行。 清理特定歷史資料。正式使用前要先確認保留政策。
完整資料庫備份 任務目標欄位會隱藏，由系統固定使用完整資料庫備份目標。 每日或每週建立完整資料庫備份，是最重要的自動化保護任務。
設計上「完整資料庫備份」不應出現在匯出或刪除的任務目標裡；只有任務類型選完整資料庫備份時，系統才會在後端固定使用備份目標。

11. API 與 ERP 匯出

API

系統提供外部 API，可查詢員工、打卡紀錄、指定區間紀錄、指定員工紀錄，也可建立 API 打卡。這些 API 需要 API Key 保護。

ERP 匯出建議

  • 若 ERP 只吃每日考勤，請固定日期、工號、姓名、班別、打卡類型、狀態與來源欄位。
  • 若 ERP 要算薪資與請假，請使用包含請假明細的模板，保留假別、請假起訖、請假時數與請假單號。
  • 匯出前先用一小段日期測試，再讓 ERP 自動提取正式檔案。
  • 自訂欄位順序後不要頻繁變更，避免 ERP 解析規則失效。

12. 首次驗收清單

  • 已可開啟 http://127.0.0.1:3000/browser/。
  • 系統管理者已登入並修改預設密碼。
  • 至少一位員工被授權為完整管理者。
  • 至少一位員工被授權為開發人員。
  • 管理者密碼與系統密碼均已修改。
  • 員工名冊已建立或匯入，卡號與工號沒有重複或空白。
  • 班別、遠端打卡 GPS 與裝置綁定政策已確認。
  • 請假假別與部門主管審核路徑已建立。
  • 員工可登入並完成一筆測試打卡。
  • 管理者可查詢考勤報表並匯出 CSV。
  • 請假核准後可在考勤匯出中看到請假紀錄。
  • 完整資料庫備份可手動建立，且最近備份清單有紀錄。
  • 自動化完整資料庫備份排程已建立。
  • 操作稽核與自動化日誌可正常查看。
  • 已將首次設定後的備份檔複製到非本機單一磁碟的位置。

# TanChin-Time-Clock-SQLite3 更新紀錄

這份文件用來保存每次功能優化、錯誤修正、資料結構調整、操作流程變更與備份流程異動。之後每次完成修改後，都要在這裡追加一筆紀錄，並且每次核心資產備份都會把這份文件一起打包。

## 備份指令

在專案根目錄執行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1"
```

預設備份輸出位置：

```text
C:\Users\nickt\OneDrive\Developer 公司系統\TanChin-Time-Clock-SQLite3 原始資料
```

備份內容包含：

- 專案原始碼與設定：主程式、瀏覽器端、桌面端、資料庫模組、匯出模組、package 設定。
- 視覺與介面資產：assets、icon、modals、browser-client、renderer-scripts。
- 開發文件與文案：docs、根目錄 txt/md 文件、研發提示語等。
- 備份工具本身：scripts。
- 營運核心資料：`%APPDATA%\TanChin-Time-Clock-SQLite3\app_data.db` 與相關 SQLite 檔。
- 使用者自訂素材：CustomSounds、ThemeImages、AuditArchives、logs。

不包含：

- `node_modules`
- `dist`
- Electron/Chromium 快取資料，例如 Cache、Code Cache、GPUCache、Network、Session Storage。

## 2026-05-12 / v1.1.47

### 新增：請假申請與雙層審核模組

- 新增請假資料結構：假別設定、部門主管審核路徑、請假申請主檔與審核步驟。
- 預設建立常用假別：事假、特休、病假、婚假、喪假、公假與其他。
- 假別保留可調整欄位，包含假別名稱、單位、是否啟用、是否有薪、是否扣額度、是否需要附件與排序。
- 員工前端新增請假申請表單，可選假別、起迄日期時間、請假時數與原因。
- 員工可查看自己的請假紀錄，並可撤回尚未管理部終審的申請。
- 若員工被設定為部門主管，員工登入後會看到「待我審核」區塊，可先行核准或駁回部門員工請假單。
- 管理者工作台新增「請假管理」分頁，主管核准後的請假單會進入管理部待複核，由管理者終審核准或駁回。
- 請假終審核准後才視為生效，狀態流程為：主管審核中 → 管理部複核中 → 已核准。

### 新增：請假設定與權限控管

- 管理者可在請假管理中維護假別設定，不需修改程式碼即可調整公司假別規則。
- 管理者可設定部門對應主管，也可用 `*` 建立預設主管路徑。
- 主管仍以員工身份登入，只能審核被設定到自己名下的請假單，不會取得管理者權限。
- 開發人員身份切換可用來測試員工、主管與管理者流程；稽核紀錄會保留操作角色。

### 優化：考勤報表整合

- 管理者考勤報表會納入已核准請假資料，來源顯示為「請假模組」。
- 報表摘要新增請假筆數，避免已核准請假被歸類為異常。
- CSV 匯出支援請假來源與請假類型，方便後續薪資或人資系統串接。

### 風險控管與後續預留

- 第一版先完成申請、主管審核、管理部終審與報表顯示，不先硬寫完整法規額度計算。
- 附件欄位與額度扣抵採資料結構預留，後續可加入病假證明、特休額度流水帳與封存策略。
- 請假申請會檢查同一員工是否已有重疊的待審或已核准請假，降低重複申請風險。
- 請假操作會寫入操作稽核，方便追查申請、撤回、主管審核與管理部終審紀錄。

## 2026-05-10 / v1.1.47

### 新增：開發人員身份切換

- 開發人員登入後可在開發人員工作台切換到管理者或指定員工頁面進行測試。
- 身份切換只允許在已登入開發人員狀態下使用；一般員工與管理者登入頁不接受系統密碼作為萬用登入密碼。
- 新增「開發人員身份切換」控制開關，放在開發人員工作台的系統設定區塊，可由系統密碼驗證後啟用或停用。
- 切換測試身份時保留原本開發人員登入狀態，並提供返回開發人員頁面的操作入口。

### 優化：開發人員工作台版面

- 將工作台標題調整為「開發者工作台」，登入狀態與身份資訊移到右上角狀態膠囊。
- 將立即任務、排程觀測、安全維護等說明內容集中到右上角「說明」按鈕內，降低主畫面資訊密度。
- 將日誌與密碼頁面拆分為「自動化日誌」、「操作稽核紀錄」與「系統設定」等頁面。
- 將變更系統密碼與開發人員身份切換整合到「系統設定」，並提高欄位密度。

### 優化：功能區塊收合與釘選

- 工作台各功能區塊標題旁新增展開/收合按鈕，進入頁面時預設收合。
- 展開/收合狀態在同一次登入操作中保留，登出再登入後才回到預設狀態。
- 新增圖釘按鈕，可設定每個區塊跨登入後預設展開或預設收合。
- 統一功能區標題列右側狀態、按鈕與展開/圖釘控制的位置。

### 優化：管理者工作台版面與說明

- 管理者工作台的人員與編號、作息與問候、主題與特效說明移到右上角「說明」按鈕內。
- 安全設定加入更完整說明，包含裝置綁定、GPS 要求、定位精度、綁定解除與風險紀錄查看。
- 員工 CSV 匯入/匯出邏輯加入右上角說明：匯出為完整員工名冊，匯入採同工號更新、同卡號更新原持卡員工、新工號且卡號不重複則新增。
- 將「匯入 CSV」與「匯出 CSV」按鈕移到「員工與名冊」區塊，放在員工筆數與圖釘控制之間。

### 新增：管理者儀表板卡片明細

- 管理者工作台上方七張統計卡片改為可點擊開啟明細彈窗。
- 「員工數量」可設定監測人員範圍：全部員工、在職員工、指定部門或手動選取員工。
- 「班別數量」顯示所有班別，並標示目前時間落在哪個班別。
- 「今日正常打卡」依監測範圍顯示已正常打卡員工與未打卡員工。
- 「今日異常打卡」顯示今日異常打卡紀錄與未成功寫入的打卡失敗稽核。
- 「響鈴排程」顯示啟用/全部排程數量，並顯示下一個響鈴排程名稱與時間。
- 「問候語」與「自訂主題」新增明細，方便查看問候語對象、主題排程與節日特效狀態。

### 風險控管與資料安全

- 管理者儀表板的監測人員範圍只影響前端儀表板統計與明細，不會修改員工名冊、打卡流程或考勤報表。
- 今日出勤狀態改以員工為單位呈現，避免原本以打卡紀錄筆數統計造成「一人多筆」的誤解。
- 異常打卡明細區分「已寫入打卡紀錄」與「失敗稽核」，避免管理者誤判資料是否已進入正式打卡紀錄。
- 後端新增管理者儀表板明細資料，前端只負責呈現與本機監測範圍設定，降低資料不一致風險。
